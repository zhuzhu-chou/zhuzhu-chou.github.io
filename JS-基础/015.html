这篇文章的意义在于牢固地记住正则匹配的元字符

# 字符匹配

# 模糊匹配

精确匹配的意义不大，正则匹配的魅力就在于模糊匹配。

横向模糊匹配。是指要匹配的长度不确定，其实现方式是使用量词。如 {m, n}

纵向模糊匹配。是指正则匹配字符串时，具体到某一位字符时，它可以不是某个确定的字符，其实现方式是使用字符组。如[abc]

# 字符组

TIP

虽叫字符组（字符类），但只是其中一个字符。

匹配1-6和a-f，可以写成下面这样

[1-6a-fG-M]
1

那我要匹配"a"、"-"、"z" 字符呢

[-az]或[az-]或[a\-z]
1

如果我不想匹配某些字符呢？

可以用到排除字符组，如

[^abc]
1

几种常见的字符组简写

\d就是[0-9]，digit
\D就是[^0-9]
\w就是[0-9a-zA-Z_]，word
\W是[^0-9a-zA-Z_]。
\s是[ \t\v\n\r\f]。表示空白符，space character
\S是[^ \t\v\n\r\f]。 非空白符。
.就是[^\n\r\u2028\u2029]。通配符。

# 量词

简写形式

{m,} 表示至少出现m次。
{m} 等价于{m,m}，表示出现m次。
? 等价于{0,1}，表示出现或者不出现。
+ 等价于{1,}，表示出现至少一次。
* 等价于{0,}，表示出现任意次，有可能不出现。

var regex = /\d{2,5}/g;

# 多选分支

一个模式可以实现横向和纵向模糊匹配。而多选分支可以支持多个子模式任选其一。

具体形式如下：(p1|p2|p3)，其中p1、p2和p3是子模式，用|（管道符）分隔。

分支结构是惰性匹配的：

var regex = /good|goodbye/g;
var string = "goodbye";
console.log( string.match(regex) ); 
// => ["good"]

# 位置匹配

# 什么是位置？

位置是相邻字符之间的位置。比如，下图中箭头所指的地方：

# 如何匹配位置?

ES5：^、$、\b、\B、(?=p)、(?!p)

ES6：(?<=p)、(?!=p)

^和$

^（脱字符）匹配开头，在多行匹配中匹配行开头。

$（美元符号）匹配结尾，在多行匹配中匹配行结尾。

看个栗子：

var result = "hello".replace(/^|$/g, '#');
console.log(result); 
// => "#hello#"
1
2
3

我们发现：位置是可以替换成字符的！

\b和\B

\b是单词边界，具体就是\w和\W之间的位置，也包括\w和^之间的位置，也包括\w和$之间的位置。

看个栗子：

var result = "[JS] Lesson_01.mp4".replace(/\b/g, '#');
console.log(result); 
// 这里注意：空格也算一个字符，所以空格和相邻的]、L之间都有一个位置
// => "[#JS#] #Lesson_01#.#mp4#"
1
2
3
4

知道了\b的概念后，那么\B也就相对好理解了。

\B就是\b的反面的意思，非单词边界。例如在字符串中所有位置中，扣掉\b，剩下的都是\B的。

具体说来就是\w与\w、\W与\W、^与\W，\W与$之间的位置。

var result = "[JS] Lesson_01.mp4".replace(/\B/g, '#');
console.log(result); 
// 这里注意：空格也算一个字符，所以空格和相邻的]、L之间都有一个位置
// => "#[J#S]# L#e#s#s#o#n#_#0#1.m#p#4"

(?=p)，其中p是一个子模式，即p前面的位置。

var result = "hello".replace(/(?=l)/g, '#');

"hello" == "" + "h" + "" + "e" + "" + "l" + "" + "l" + "o" + "";

# 引用分组

这是括号一个重要的作用，有了它，我们就可以进行数据提取，以及更强大的替换操作。

而要使用它带来的好处，必须配合使用实现环境的API。

以日期为例。假设格式是yyyy-mm-dd的，我们可以先写一个简单的正则：

var regex = /\d{4}-\d{2}-\d{2}/;

var regex = /(\d{4})-(\d{2})-(\d{2})/;

# 反向引用

除了使用相应API来引用分组，也可以在正则本身里引用分组。但只能引用之前出现的分组，即反向引用。

比如要写一个正则支持匹配如下三种格式：

2016-06-12

2016/06/12

2016.06.12

最先想到的正则是：

var regex = /\d{4}(-|\/|\.)\d{2}(-|\/|\.)\d{2}/;
var string1 = "2017-06-12";
var string2 = "2017/06/12";
console.log( regex.test(string1) ); // true
console.log( regex.test(string2) ); // true
1
2
3
4
5

其中/和.需要转义。虽然匹配了要求的情况，但也匹配"2016-06/12"这样的数据。

var string4 = "2016-06/12";
console.log( regex.test(string4) ); // true
1
2

假设我们想要求分割符前后一致怎么办？此时需要使用反向引用：

var regex = /\d{4}(-|\/|\.)\d{2}\1\d{2}/;
var string4 = "2016-06/12";
console.log( regex.test(string4) ); // false
1
2
3

\1表示引用之前的分组(-|\/|.)，不管它匹配到什么（比如-），\1都匹配那个同样的具体某个字符。

我们知道了\1的含义后，那么\2和\3的概念也就理解了，即分别指代第二个和第三个分组。

# 非捕获分组

之前文中出现的分组，都会捕获它们匹配到的数据，以便后续引用，因此也称他们是捕获型分组。

如果只想要括号最原始的功能，但不会引用它，即，既不在API里引用，也不在正则里反向引用。此时可以使用非捕获分组(?:p)，例如本文第一个例子可以修改为：

var regex = /(?:ab)+/g;
var string = "ababa abbb ababab";
console.log( string.match(regex) ); 
// => ["abab", "ab", "ababab"]
