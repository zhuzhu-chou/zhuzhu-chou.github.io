  • 减少请求数量

    • 文件合并（Http/1.1）

    • 将 JS 和 CSS 文件外链

      一些较为公用的JS和CSS，我们可以使用外链的形式,譬如我就是从Google外链来的Jquery文件，如果我的浏览者在浏览别的使用了这个外链文件的网站时已经下载并缓存了这个文件，那么他在浏览我的网站的时候就不需要再进行下载了。

    • 图片处理

      • 雪碧图
      • Base64
      • Iconfont
      <--! <img src="xxx"> 会很卡 -->
内存中先加载，然后当内存加载完毕时，再把内存中的数据填充到我们的 dom 元素中，这样能够快速的去反应

<script type="text/javascript">
			
		window.onload = function(){
			
			var img = new Image();
			img.onload = function(){
				alert('加载完毕')
				document.getElementById('bigImg').src = this.src;
			}
			img.src = "xxx.jpg";
		}	 
</script>

  • 优化资源加载

    • 资源加载位置
      • CSS文件放在head中
      • JS文件放在body底部
      • 处理页面、处理页面布局的JS文件放在head中，如babel-polyfill.js文件、flexible.js文件
      • body中间尽量不写style标签和script标签
    • 资源加载时机
      • 异步 script 标签
      • 模块按需加载。如 ES 的 import()，动态导入
      • 资源懒加载与资源预加载

  • 渲染优化

    • 样式设置
    • 渲染层
      • 动画硬件加速
      • 需要多次重绘，absoulte 脱离文档流，减少重绘区域
    • DOM 优化
      • 缓存 dom。查询 DOM 比较耗时。
      • 减少 DOM 深度，尽量简洁
      • 离线更新 DOM、DOM 读写分离等减少重绘和回流
      • 事件代理
      • 防抖和节流
      • 及时清理环境，以防内存泄露（可以使用 WeakMap）

  • 性能更好的 API

    最重要的是要考虑浏览器兼容性

    • 不同选择器也有性能差异
    • 使用 requestAnimationFrame 来替代 setTimeout 和 setInterval
    • 使用 IntersectionObserver 来实现图片可视区域的懒加载
    • web worker

  • webpack 优化（构建优化）

    • 打包公共代码

      optimization.splitChunks 和 optimization.runtimeChunk插件支持

    • 动态导入和按需加载

    • 剔除无用代码

      tree shaking。依赖于 ES2015 模块系统中的静态结构特性，如 import 和 export。

      JS 的 tree shaking 主要通过uglifyjs插件来完成，CSS 的tree shaking主要通过purify CSS来实现的。
