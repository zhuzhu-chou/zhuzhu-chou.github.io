http是一个不保存状态的协议，Cookie 就是一种浏览器管理状态的一个文件（小型文本文件，4KB），且不可跨域 ，以文件方式存储于客户端磁盘（在本地文件夹中也可以找到），当请求某接口的时候（如www.juejin.im/post），浏览器会自动携带符合domain和path的Cookie（domain为.juejin.im并且path=/post的cookie）都携带在Request Headers 上

Cookie原理：

![](https://zhuzhu-chou.github.io/images/2020-03-17 14-41-32屏幕截图.png)

Cookie的属性：

  • **name：**一个域名下绑定的cookie，name不能相同，相同的name的值会被覆盖掉

  • **value：**关于对cookie值是否编码的说法不一，有的说仅是 name/value 中的三种符号—分号、逗号、空格编码

  • domain：Cookie 绑定的域名，如果没有设置，就会自动绑定到执行语句的当前域，还有值得注意的点

  • **path：**匹配路由，如把 path=/blog; domain=.baidu.com，就是给www.baidu.com/blog绑定 Cookie

  • Cookie 有效期属性

    • **Max-Age：**单位为秒，设置正数，cookie会在Max-age秒后被删除；负数表示临时存储，不会生成cookie文件，只会在浏览器内存中，一旦浏览器关闭，cookie也会消失；设置0，因为cookie机制本身没有设置删除cookie，失效的cookie会被浏览器自动从内存中删除，所以，它实现的就是让cookie失效。

    • **Expires：**时间结点绝对值，被Max-Age代替了。

  • 安全相关

    • secure：当这个属性设置为true时，此Cookie只会在https和SSL/TLS等安全协议下传输

    • **HttpOnly：**如果这个属性设置为 true，就不能通过 js 脚本来获取 Cookie 的值，能有效的防止xss攻击![](/home/boygreen/图片/2020-03-17 15-54-54屏幕截图.png)

    • SameSite： 可以让 Cookie 在跨站请求时不会被发送，从而可以阻止跨站请求伪造攻击（CSRF）。

      先说明一下同站/跨站（和第一方/第三方等价），只要两个URL的 eTLD + 1 相同，即为同站，eTLD 表示有效顶级域名，如：.com、.github.io。eTLD + 1 则表示有效顶级域名 + 二级域名，如：.taobao.com 。跨站跟跨域相比更为宽松。

      SameSite 的值：

      Chrome80 后默认是 Lax，之前都是 None。

      • Strict

        当前网页 URL 与请求目标 URL 完全一致，浏览器才允许携带 Cookie 。

        这个规则过于严格，可能造成非常不好的用户体验。比如，当前网页有一个 GitHub 链接，用户点击跳转就不会带有 GitHub 的 Cookie，跳转过去总是未登陆状态。

      • Lax

        和 Strict 相比，稍稍宽松一些，大多数情况不发送第三方 Cookie ，但是导航到目标网址的 GET 请求除外。导航到第三方网址的 GET 请求只包含三种：链接、预加载请求、GET 表单。

      • None

        无论是否跨站都会携带 Cookie 发送

      ​ 除了表格这些还有 script Lax 情况下也不发送 Cookie，jsonp 的请求也会受影响。

      注意

      • HTTP 接口不支持 SameSite=None;

        如果要想在 Cookie 加 SameSite: None; 要同时加上 Secure 属性

      • 需要 UA 检测，部分浏览器能加 SameSite=None

        所以服务端必须在下发 Set-Cookie 响应头时进行 User-Agent 检测，对这些浏览器不下发 SameSite=none 属性

**注意：**要设置多个cookie时，得多写几个Set-Cookie

Cookie 的缺点：

  • 存储安全：HTTP 明文传输，安全性一般，可以采用 Secure，或者采用 Session、Token 技术方案
  • 大小：只有4KB，和本地存储（5MB）比起来小很多
  • 增加请求大小。

如何封装一个操作 Cookie 的 API

// 客户端获取、设置、移除cookie
cookieAPI = {
  // 正则取 Cookie 
  getCookie(name) {
    const arrCookie = document.cookie.replace(/[ ]/g, "").split(';')
    for (let i = 0; i < arrCookie.length; i++) {
      const arr = arrCookie[i].split('=')
      if (name === arr[0]) {
        return arr[1]
      }
    }
    return ''
  },
  setCookie(name, value, time) {
    const time = time * 24 * 3600 * 1000
    document.cookie = `${name}=${value}; Max-Age=${time}`
  },
  removeCookie(name) {
    const value = this.getCookie(name)
    document.cookie = `${name}=${value}; Max-Age=0`
  },
}
