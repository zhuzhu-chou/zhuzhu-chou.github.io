    工作区管理

    • git add ：到暂缓区
    • git rm/mv：删除或移动文件
    • git checkout -- readme.txt：丢弃工作区的修改。

    工作区状态

    • git status：查看工作区状态。
    • git diff：如果 git status 提示你工作区发生了改变，git diff 可以用来查看发生了哪些改动。git diff HEAD -- readme.txt可以用来查看工作区和版本库里面最新版本的区别。

    版本回退

    • git reflog：用来查看命令历史，可以取到你需要的commit_id。
    • git reset --hard commit_id：回退到具体的版本。git reset --hard HEAD^回退到上一个版本。

    分支管理

    • git commit：提交到本地库

    • git push：

    • git fetch：从远程主机拉取最新内容

    • git pull : git fetch + git merge

    • git merge：假设现在在 master 分支，执行git merge dev表示把dev分支的工作成果合并到master分支上。

    • git rebase：

      假设你现在基于远程分支origin，创建一个叫mywork的分支。

      $ git checkout -b mywork origin
      1

      现在我们在这个分支做一些修改，然后生成两个提交(commit)。

      $ git checkout mywork
$ git rebase origin
      1
      2

      这些命令会把你的mywork分支里的每个提交(commit)取消掉，并且把它们临时保存为补丁(patch)(这些补丁放到.git/rebase目录中)，然后把mywork分支更新到最新的origin分支，最后把保存的这些补丁应用到mywork分支上。

      当mywork分支更新之后，它会指向这些新创建的提交(commit)，而那些老的提交会被丢弃。 如果运行垃圾收集命令 git gc，这些被丢弃的提交就会删除。

      现在我们可以看一下用合并(merge)和用rebase所产生的历史的区别：

      在rebase的过程中，也许会出现冲突(conflict)。在这种情况，Git会停止rebase并会让你去解决冲突；在解决完冲突后，用git add命令去更新这些内容的索引(index)，然后，你无需执行 git commit，只要执行:

      $ git rebase --continue
      1

      这样git会继续应用(apply)余下的补丁。

      在任何时候，你可以用--abort参数来终止rebase的行动，并且mywork 分支会回到rebase开始前的状态。

      $ git rebase --abort
      1

    • git checkout：git checkout -b dev，加上-b参数表示创建dev分支，然后切换到dev分支。

      git checkout dev 表示切换到 dev 分支。

    • git branch：列出所有分支。

    # git revert 和 git reset

    这两个命令都用于撤销提交（commit 过）

    先看看 git reset 版本回退怎么操作

    1.查找需要回退到的版本号

    $ git log
    1

    2.回退

    $ git reset --hard 版本号
    1

    3.push

    $ git push -f
    1

    git revert

    git revert是用于“反做”某一个版本，以达到撤销该版本的修改的目的。

    比如，我们commit了三个版本（版本一、版本二、 版本三），突然发现版本二不行（如：有bug），想要撤销版本二，但又不想影响撤销版本三的提交，就可以用 git revert 命令来反做版本二，生成新的版本四，这个版本四里会保留版本三的东西，但撤销了版本二的东西。如下图所示：

    适用场景： 如果我们想撤销之前的某一版本，但是又想保留该目标版本后面的版本，记录下这整个版本变动流程，就可以用这种方法。

    # 具体操作：

    举个例子，现在库里面有三个文件：READ.md、text.txt、text2.txt。 1. 查看版本号：

    $ git log
    1

    也可以通过github网站图形化界面查看版本号：

    找到需要"反做"的版本号。

    2. 反做：

    （1）反做 revert

    $ git revert -n 8b8961
    1

    （2）解决可能的冲突

    手动解决后 git add 文件名

    $ git commit -m ''
    1

    3.推送上去：

    $ git push
    1

    此时查看仓库的文件，剩下两个：READ.md、text2.txt
