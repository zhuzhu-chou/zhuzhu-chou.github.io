# 常见的术语

根域、顶级域、二级域

权威 DNS：

权威 DNS 即最终决定域名解析结果的服务器。

递归 DNS：

递归 DNS 即 Local DNS，它没有域名解析结果的决定权，但代理了用户向权威 DNS 获取解析结果的过程。递归 DNS 上有缓存模块，当目标域名存在缓存结果并且 TTL 未过期时（每个域名都有 TTL，即有效生效时间，若解析结果缓存的时间超过　TTL，需要重新向权威 DNS 获取解析结果），递归 DNS 会返回缓存结果，否则，递归 DNS 会一级一级地查询各个层级域名的权威 DNS直至获取完整域名的解析结果。

公共 DNS

递归 DNS 的一种特例，它是一种全网开放的递归 DNS 服务，二传统的递归 DNS 信息一般有 ISP 分发给用户。一个比较典型的公共 DNS 即 Google 的8.8.8.8。

TIP

在实际的使用过程中，我们通常不需要手工指定自己的Local DNS地址。运营商会通过DHCP协议在系统网络初始化阶段将Local DNS地址分配给我们的计算机。当我们需要使用公共DNS服务时，我们就必须手工指定这些服务的地址。以Linux为例，我们可以通过在'/etc/resolv.conf'中添加Local DNS地址项来改变本机Local DNS的地址。

# DNS流程流程

以访问www.taobao.com为例

  1. 客户端向 Local DNS 发起域名解析请求
  2. Local DNS 在获取到域名解析请求后首先从 Root hints （根提示）获取根域名服务器的地址（Root hints 包含了互联网中根域名服务器的地址信息）
  3. 之后 Local DNS 向根域名服务器发起 DNS 解析请求，根域名服务器返回 com 顶级域名服务器地址
  4. 随后 Local DNS 向 com 域名服务器发请求，得到 taobao.com 二级域名服务器地址，
  5. Locasl DNS 向 taobao.com 二级域名服务器发起请求，最终获得 www.taobao.com的 IP 地址信息
  6. Local DNS 将查询结果缓存并返回给客户端
